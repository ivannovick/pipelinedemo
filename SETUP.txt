Architecture (stream-first)
----------------------------
1) Airflow DAG `gdrive_publish_to_stream` uses dlt to read PDFs from Google Drive and publishes
   JSON envelopes to a RabbitMQ *stream* (port 5552). It does not parse PDFs or touch Postgres/disk.

2) Airflow DAG `pdf_stream_batch_consumer` runs on a schedule (default: every 1 minute). Each run
   subscribes to the stream for a bounded time, processes messages in the callback (decode PDF,
   pypdf extract, write ./data/pdfs, INSERT Postgres), and stores consumer offsets on the broker.
   This replaces the old long-running `stream-processor` container.

3) Postgres holds extracted_text plus source_stream + stream_offset (idempotency) and paths to files.

4) Airflow DAG `dbt_run_pdf_archive` (transform only, not tied to RMQ): dbt-postgres in the Airflow
   image; project at dbt/pdf_archive → /opt/dbt/pdf_archive. Runs `dbt run` on its own schedule
   (default every 60 minutes; DBT_RUN_SCHEDULE_MINUTES in .env). Builds dbt_analytics models
   (stg_pdf_documents, pdf_documents_enriched, pdf_documents_fts + GIN). Manual run:
     docker compose exec airflow bash -lc "mkdir -p /tmp/dbt-target /tmp/dbt-logs && cd /opt/dbt/pdf_archive && dbt run --project-dir /opt/dbt/pdf_archive --profiles-dir /opt/dbt/pdf_archive --target-path /tmp/dbt-target --log-path /tmp/dbt-logs"

Postgres column for your PC archive path
----------------------------------------
- local_pc_archive_path (NOT NULL): where the PDF file lives relative to your project, or an absolute path.
  - Default: "data/pdfs/<stored_file_name>" — resolve as <your repo>/data/pdfs/... on your machine (same as the Compose bind mount).
  - Optional: set PDF_HOST_BASE_DIR in .env to the absolute path of your data/pdfs folder (e.g. C:\Users\you\...\airbyteapp\data\pdfs); then this column stores the full path including the filename.

Startup / latency
-----------------
- Four containers: postgres, rabbitmq, airflow, workflow-viz (optional diagram UI).
- Ingest from Drive can run anytime; PDFs are usually processed within about one schedule interval
  (default 1 minute) plus STREAM_BATCH_POLL_SECONDS per batch run. Tune STREAM_BATCH_POLL_SECONDS
  and STREAM_BATCH_MAX_MESSAGES in .env if runs overlap or time out.
- For catch-up after long downtime, set STREAM_READ_OFFSET=first once (then back to next) or use a
  new STREAM_CONSUMER_NAME.

Volumes
-------
- ./data/pdfs: PDF binaries written by the batch consumer (bind mount into Airflow).
- postgres-data: Postgres data.
- ./tools, ./testdata: read-only into Airflow for local publish helper (see TESTING.txt).
- ./dbt/pdf_archive: read-only into Airflow for dbt (used by pdf_stream_batch_consumer).

Services (containers)
---------------------
- postgres, rabbitmq, airflow (includes dbt CLI for the dbt DAG), workflow-viz (Clarity UI;
  default http://localhost:3456).

List PDFs from your Google account (local PC)
----------------------------------------------
See tools/DRIVE_LIST_SETUP.txt — script tools/list_drive_pdfs.py supports OAuth (personal Gmail)
or the same service account as dlt/service_account.json / dlt/secrets.toml.

Setup
-----
1. Google Drive: put your GCP service account JSON at dlt/service_account.json (same file
   as "Create key" in Cloud Console), OR copy secrets.toml.example to dlt/secrets.toml
   and fill [sources.filesystem.credentials]. Share the Drive folder with the SA email.
2. .env from .env.example.
3. If you already initialized Postgres with an older schema, remove the postgres-data volume,
   or run postgres/migrations/001_local_pc_archive_path.sql, or recreate the table from 01-schema.sql.
4. docker compose up --build
5. Enable DAGs `pdf_stream_batch_consumer` (stream → Postgres), `dbt_run_pdf_archive` (dbt transforms),
   and `gdrive_publish_to_stream` (Drive → stream). Rebuild airflow after pulling: docker compose build airflow.

Message format (stream body)
----------------------------
Default from publisher: JSON UTF-8 with pdf_base64, original_file_name, source_file_url.
The batch consumer also accepts raw PDF bytes (body starts with %PDF) for other producers.

Query
-----
  docker compose exec postgres psql -U pdfuser -d pdfarchive -c "SELECT source_stream, stream_offset, original_file_name, left(extracted_text,60) FROM pdf_documents;"

Notes
-----
- Very large PDFs may require raising RabbitMQ stream / message limits.
- Airflow metadata remains SQLite under ./airflow-sqlite (PoC).
