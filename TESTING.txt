Local test (no Google Drive)
----------------------------
1. docker compose up -d --build
2. Wait until RabbitMQ and Airflow are healthy (Airflow logs show webserver/scheduler up).
3. Enable DAG `pdf_stream_batch_consumer` in the UI (or it will run on schedule every minute).
4. Publish the sample PDF (same envelope format as the Airflow DAG), using the Airflow container:

   docker compose exec airflow python /opt/airflow/tools/publish_stream_once.py /opt/airflow/testdata/sample.pdf

5. Within about one minute (next batch run), confirm:
   - docker compose logs airflow --tail 80   (look for pdf_stream_batch_consumer / "Processed stream")
   - docker compose exec postgres psql -U pdfuser -d pdfarchive -c "SELECT * FROM pdf_documents;"
   - A new .pdf under ./data/pdfs

   To run the consumer immediately instead of waiting for the schedule, trigger DAG
   `pdf_stream_batch_consumer` once from the Airflow UI.

Regenerate testdata/sample.pdf (optional)
-----------------------------------------
  docker run --rm -v "%CD%/tools:/w" -v "%CD%/testdata:/out" python:3.12-slim sh -c "pip install -q fpdf2 && python /w/gen_sample_pdf.py /out/sample.pdf"

Test with one PDF on Google Drive
-----------------------------------
1. Google Cloud: enable Drive API, create a service account, download JSON key.
2. Put the GCP service account JSON at dlt/service_account.json, or copy dlt/secrets.toml.example
   to dlt/secrets.toml and fill [sources.filesystem.credentials].
3. In Drive, create or pick a folder and share it with the service account email (Viewer is enough).
4. Upload exactly one PDF into that folder (or ensure only one matches **/*.pdf).
5. Set folder URL in .env:
     GDRIVE_BUCKET_URL=gdrive://<FOLDER_ID>/
   The folder ID is the long id in the Drive URL when you open the folder in a browser.
6. docker compose up -d --build
7. Open Airflow http://localhost:8080 — get admin password from:
     docker compose logs airflow 2>&1 | findstr /i password
8. Enable DAG `pdf_stream_batch_consumer` and DAG `gdrive_publish_to_stream`; trigger
   `gdrive_publish_to_stream` once (or wait for schedule).
9. After up to ~1 minute, check Postgres and ./data/pdfs as above (or trigger `pdf_stream_batch_consumer`).

If the DAG fails on credentials, fix dlt/service_account.json (or secrets.toml) and bucket URL until dlt can list the folder.
